# API切换器

一个强大的 API 管理和切换工具，支持多种 AI API 提供商，并具有智能错误自动恢复功能。

## ✨ 主要功能

### 1. API 管理
- 支持多个 API 提供商（Anthropic Claude、OpenAI、DeepSeek V4、Kimi、GLM、自定义 OpenAI-compatible）
- 快速切换不同的 API 配置
- 安全的密钥管理
- 配置导入/导出
- 在 Profile 编辑器中测试连接、刷新模型列表、查看 provider 专属提示

### 2. 自动续跑功能
- 自动检测任务是否完成
- 智能续跑未完成的任务
- 支持 Claude Code 和 Codex CLI
- 可配置的续跑策略

### 3. 错误自动恢复 ⭐
- **智能识别 10 种 API 错误类型**
  - 内容超长
  - 速率限制
  - 认证错误
  - 权限错误
  - 配额超限
  - 模型过载
  - 超时错误
  - 网络错误
  - 服务器错误
  - 无效请求

- **5 种自动恢复策略**
  - 压缩并继续（内容超长）
  - 等待后重试（速率限制）
  - 指数退避重试（服务器问题）
  - 友好提示（认证/权限/配额）
  - 中止执行（无效请求）

- **详细的错误统计**
  - 可视化统计卡片
  - 按错误类型分析
  - 错误时间线
  - 导出详细报告

## 🚀 快速开始

### 下载和安装

1. 下载 `API切换器.exe`
2. 双击运行即可，无需安装

### 首次使用

1. **添加 API 配置**
   - 点击"添加配置"按钮
   - 选择 API 提供商
   - 输入 API 密钥和其他配置
   - 保存

2. **切换 API**
   - 在配置列表中选择要使用的配置
   - 点击"应用"按钮
   - 系统会自动更新相应的配置文件

3. **启用错误自动恢复**（推荐）
   - 切换到"通用设置" Tab
   - 找到 Claude Code 或 Codex CLI
   - 点击"设置"按钮
   - 勾选"启用错误自动恢复"
   - 设置最大恢复次数为 3
   - 点击"保存"

### 使用场景

#### 场景 1: 长时间对话
当使用 Claude Code 进行长时间对话时，如果遇到内容超长错误，系统会自动执行压缩命令并继续，无需手动干预。

#### 场景 2: 频繁调用
当 API 请求过于频繁触发速率限制时，系统会自动等待指定时间后重试，确保任务顺利完成。

#### 场景 3: 服务器不稳定
当遇到服务器繁忙或超时错误时，系统会使用指数退避算法自动重试，提高成功率。

## 💡 使用技巧

### 1. 配置管理
- 使用有意义的配置名称，方便识别
- 定期备份配置（导出功能）
- 为不同项目创建不同的配置

### 2. 错误恢复
- 建议启用错误自动恢复功能
- 设置合理的最大恢复次数（推荐 3-5 次）
- 定期查看错误统计，了解 API 使用情况

### 3. 自动续跑
- 根据任务类型选择保守模式或激进模式
- 自定义未完成模式和阻塞模式
- 配合错误恢复使用，效果更佳

## 📊 错误统计功能

点击"错误统计"按钮可以查看：
- 总错误数、成功恢复数、恢复成功率
- 按错误类型的详细统计
- 最近的错误详情
- 导出详细报告

## ⚠️ 注意事项

### 安全性
- API 密钥存储在本地，请妥善保管
- 不要将配置文件分享给他人
- 定期更换 API 密钥

### 隐私
- 本应用不会上传任何数据到服务器
- 所有配置和日志都存储在本地
- 不会收集用户信息

### 兼容性
- 确保 Claude Code 或 Codex CLI 已正确安装
- 配置文件路径需要正确
- 某些杀毒软件可能误报，请添加信任

## 🐛 故障排除

### 问题 1: 应用无法启动
**解决方法**:
- 检查是否被杀毒软件拦截
- 尝试以管理员身份运行
- 查看日志文件（在应用数据目录）

### 问题 2: 配置无法保存
**解决方法**:
- 检查文件权限
- 确保配置目录存在
- 检查磁盘空间

### 问题 3: 错误恢复未触发
**解决方法**:
- 确认已启用错误恢复功能
- 检查 Hook 脚本是否正确安装
- 运行验证脚本（如果有源码）

### 问题 4: API 切换不生效
**解决方法**:
- 检查配置文件路径是否正确
- 确认 API 密钥有效
- 重启相关应用（Claude Code/Codex CLI）
